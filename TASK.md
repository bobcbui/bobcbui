# UI 重构建议

## 总体方向

将网站明确拆成两种体验：

- 首页是轻量、快速、移动端友好的个人导航页。
- 其他页面是专注阅读的博客/内容页。

保留现有页面、链接和文案，通过统一设计规范和 Vue 组件逐步重构，不一次性推翻全部代码。

## 粉红主题规范

- 主题使用低饱和粉红色，整体保持现代、克制和清爽。
- 页面背景使用暖白或极浅粉，正文继续使用深灰色，保证阅读对比度。
- 粉红色主要用于按钮、链接、焦点框、当前状态和少量图标背景。
- 大面积内容卡片保持白色，不使用高饱和渐变、霓虹效果或大面积纯粉背景。
- Hover 状态使用稍深的粉红色，Focus 状态使用半透明粉色外框。

建议颜色：

```css
--color-bg: #fff8fa;
--color-bg-elevated: #ffffff;
--color-primary: #db4b78;
--color-primary-hover: #c93868;
--color-primary-soft: #fce7ef;
--color-text-main: #2b2025;
--color-text-muted: #74636a;
--color-border: #f0dce4;
```

## 移动端优先原则

- 所有页面先以 360px 宽度完成布局，再逐步增强平板和桌面端。
- 点击区域不小于 44×44px，按钮和链接之间保留足够间距。
- 正文、导航标题和搜索框不使用固定宽度，长文本允许自然换行。
- 手机端优先展示核心内容，次要入口使用折叠或展开，不堆满首屏。
- 避免依赖 Hover 才能使用的交互，所有功能必须支持触摸操作。
- 控制资源体积与脚本执行时间，弱网环境下仍能先看到基础内容。

## P0：优先处理

### 1. 建立统一的设计变量

- 将颜色、字号、间距、圆角、阴影、内容宽度和响应式断点集中为 CSS Variables。
- 所有组件统一使用粉红主题变量，不在组件内部散落独立色值。
- 首页和内容页共享基础字体、背景、链接状态与焦点状态。
- 减少目前样式表中重复或互相覆盖的规则，避免后续调整一个区域影响其他页面。

建议基准：8px 间距体系、12px 卡片圆角、正文最大宽度 760px、导航最大宽度 1120px。

### 2. 重构首页信息层级

- 第一层：站点名称和搜索框，保持同一行。
- 第二层：一句主标题和一句说明，控制高度，不做大面积 Hero。
- 第三层：导航分类卡片，桌面三列、平板两列、手机单列。
- 每个分类默认展示 4～6 个入口，数量过多时使用“展开全部”，避免单张卡片过长。
- 搜索应同时支持筛选本站导航项；是否跳转百度作为次级操作，而不是唯一搜索方式。

### 3. 统一内容页为文章模板

- 所有普通页面使用同一套 `ArticleLayout`：页头、文章标题、正文、更新时间/返回首页区域。
- 正文保持窄栏阅读，合理控制段落行长、标题间距、列表缩进和代码块横向滚动。
- 手机端移除厚重阴影，减小外层留白，让正文获得更多可用宽度。
- 页面原有文本保持不变，只统一结构与展示方式。

## P1：组件化重构

### 4. 拆分 Vue UI 组件

- `SiteHeader`：站点名称、桌面导航、移动端菜单、首页搜索。
- `NavSearch`：导航关键词过滤、清空按钮、键盘操作。
- `NavGroup`：分类标题、说明、入口列表、展开状态。
- `ArticleLayout`：内容页统一阅读容器。
- `AppShell`：页面最大宽度、背景、页头和主内容区域。

组件只负责展示和交互，导航数据继续集中维护，避免在 HTML 和组件中重复写链接。

### 5. 扩展 `core.js` 的职责边界

- 统一导出 Vue API 和组件挂载方法。
- 增加通用的 `mountPage()`，集中处理错误捕获和挂载状态。
- 增加导航数据加载函数，例如 `loadNavigation()`，把 XML 解析从 `index.html` 移出。
- 不要把业务组件模板直接放进 `core.js`，Core 只提供基础能力。

建议目录：

```text
static/
├── core.js
├── data/
│   └── navigation.js
├── components/
│   ├── SiteHeader.js
│   ├── NavSearch.js
│   ├── NavGroup.js
│   └── ArticleLayout.js
└── styles/
    ├── tokens.css
    ├── base.css
    ├── components.css
    └── pages.css
```

## P2：体验完善

### 6. 补齐交互与无障碍状态

- 为键盘用户提供清晰的 `:focus-visible` 样式。
- 搜索框支持 `/` 快捷键聚焦、`Esc` 清空。
- 当前页面对应的页头导航增加选中状态。
- 移动端菜单打开后支持点击链接自动关闭，并避免背景误滚动。
- 加载导航时使用与最终卡片尺寸接近的骨架屏，减少布局跳动。
- 尊重 `prefers-reduced-motion`，降低不必要的悬浮和位移动画。

## 推荐实施顺序

1. 以 360px 为基准整理 CSS Variables、基础样式和触摸区域。
2. 提取导航数据加载逻辑，拆出 `NavGroup` 与 `NavSearch`。
3. 将内容页统一到 `ArticleLayout`。
4. 扩展 768px 和 1440px 布局，优化平板与桌面端空间利用。
5. 最后补充快捷键与细节动画。

## 验收标准

- 360px、768px、1440px 三种宽度下没有横向滚动或内容遮挡。
- 触摸按钮和主要链接的可点击区域不小于 44×44px。
- 手机端无需 Hover 即可完成搜索、打开导航和访问全部入口。
- 首页首屏能看到搜索、简介和至少一组完整导航。
- 导航项可以通过关键词即时筛选，并可完全使用键盘操作。
- 内容页正文宽度统一，原有页面和文本没有丢失。
- Vue 组件只通过 `core.js` 使用公共能力，导航链接没有重复维护。
